## 一、项目目标与范围

### 1.1 核心目标

基于三方讨论共识，为agent-chat-hub添加以下核心能力：

1. **角色系统** - 通过Prompt动态定义不同角色，无需硬编码
2. **团队协作** - 支持多角色组建团队，协同完成复杂任务
3. **Coordinator总管** - 唯一用户接口，负责意图判定和答复整合
4. **显式@路由** - 用户可通过@指定角色交流，被指定角色必须响应
5. **自主路由** - 角色可自主判定交流对象，受约束条件限制

### 1.2 设计原则

**P1 - 兼容性保护**（Codex核心关切）：
- 未启用团队模式时，保持现有单Agent行为
- 无@普通对话行为不受影响
- 现有ResponseCoordinator功能继续工作

**P2 - 确定性路由**（Gemini架构要求）：
- 显式@优先于自主路由和总管分派
- 路由优先级和去重规则明确定义
- 多角色@采用Scatter-Gather模式

**P3 - 可控性边界**（Codex安全关切）：
- 自主路由受轮次、预算、权限、循环检测约束
- 被@角色可因权限/能力不足/超时明确拒绝
- 所有路由决策可审计

### 1.3 非目标（本次不包含）

- ❌ 跨会话的持久化团队配置（未来Phase 6）
- ❌ 动态角色能力学习和自适应（未来Phase 7）
- ❌ 角色间的并发事务和分布式锁（当前假设单进程）

### 2.2 数据模型扩展

#### RoleConfig（新增）

```python
@dataclass
class RoleConfig:
    """角色配置"""
    role_id: str              # 角色唯一标识
    name: str                 # 显示名称
    base_model: str           # 底层模型ID (引用ModelConfig)
    system_prompt: str        # 角色定义Prompt
    is_coordinator: bool      # 是否为总管角色
    capabilities: List[str]   # 能力标签列表
    max_tokens: int = 4096    # 单次响应token限制
    temperature: float = 1.0  # 温度参数
    priority: int = 100       # 优先级（保持兼容）
    active: bool = True       # 是否激活（保持兼容）
```

#### MessageEnvelope（新增）

```python
@dataclass
class MessageEnvelope:
    """消息信封 - 扩展路由信息"""
    content: str                      # 消息内容
    explicit_targets: List[str]       # @显式指定的角色ID列表
    source_role: Optional[str]        # 发送者角色ID
    routing_mode: RoutingMode         # 路由模式
    parent_message_id: Optional[str]  # 父消息ID（追踪对话链）
    max_rounds: int = 5               # 自主路由最大轮次
    budget_tokens: int = 50000        # 成本预算
```

#### RoutingMode（新增枚举）

```python
class RoutingMode(Enum):
    """路由模式"""
    EXPLICIT = "explicit"      # 显式@指定
    AUTO = "auto"              # 总管自动分派
    INTERNAL = "internal"      # 角色间自主通信
    BROADCAST = "broadcast"    # 广播（多角色@）
```

## 三、路由优先级与冲突解决

### 3.1 优先级规则（解决Codex阻塞问题2）

**规则定义**：

```
Priority 1 (最高): 显式@指定 (EXPLICIT / BROADCAST)
  - 用户输入包含 @role_id
  - 直接路由到指定角色
  - 绕过Coordinator意图识别

Priority 2 (中等): Coordinator分派 (AUTO)
  - 无显式@的用户输入
  - Coordinator分析意图后决定路由
  - 可能选择单个或多个角色

Priority 3 (最低): 角色自主通信 (INTERNAL)
  - 角色间互相调用
  - 需通过AutoRoutingGuard检查
  - 受轮次、预算、循环约束
```

### 3.2 去重规则

**场景**: 同一角色被多种路由方式选中

**规则**:
1. 显式@ > 自动分派 > 自主通信
2. 高优先级路由成功后，低优先级路由自动取消
3. 同优先级重复选中时，仅调用一次
4. 记录去重日志供审计

**示例**:
```
用户输入: "@codex @gemini 请设计API，Coordinator也认为需要codex"
结果: codex仅被调用一次（显式@），Coordinator的自动分派被去重
```

```python
def append_message(self, envelope: MessageEnvelope):
    """
    显示所有角色间通信，确保完全透明
    """
    
    # 格式化发送者和接收者
    sender = envelope.source_role or "user"
    target = envelope.target_role or "user"
    
    if envelope.routing_mode == RoutingMode.EXPLICIT:

        icon = "🎯"

        self.display_full_message(prefix, envelope.content)

        if envelope.source_role == "coordinator":
            icon = "📤"
            prefix = f"{icon} Coordinator → @{target}"
            self.display_full_message(prefix, envelope.content)
        else:

            icon = "📥"
            prefix = f"{icon} @{sender} → @coordinator"
            self.display_full_message(prefix, envelope.content)
    
    elif envelope.routing_mode == RoutingMode.INTERNAL:

        prefix = f"{icon} @{sender} → @{target}"
        self.display_full_message(prefix, envelope.content)

        # 普通用户消息
        self.display_full_message("💬 User", envelope.content)

        self.chat_display.renderable + formatted

    self.chat_container.scroll_end(animate=False)
```

### 5.2 Scatter-Gather实现细节（Gemini方案）

#### 5.2.1 核心代码结构

```python
import asyncio
from typing import List, Dict, Optional
from dataclasses import dataclass

@dataclass
class ScatterResult:
    role_id: str
    success: bool
    response: Optional[str]
    latency_ms: int
    error: Optional[str]

class ScatterGatherExecutor:
    def __init__(self, timeout: int = 30):
        self.timeout = timeout
    
    async def execute(
        self, 
        targets: List[str], 
        message: str
    ) -> List[ScatterResult]:
        """并行调用多个角色，收集响应"""
        
        # Scatter阶段：创建并发任务
        tasks = [
            self._call_role(role_id, message) 
            for role_id in targets
        ]
        
        # Gather阶段：等待所有任务（带超时）
        results = await asyncio.gather(
            *tasks, 
            return_exceptions=True
        )
        
        # 处理结果
        return [
            self._wrap_result(role_id, result)
            for role_id, result in zip(targets, results)
        ]
    
    async def _call_role(
        self, 
        role_id: str, 
        message: str
    ) -> str:
        """调用单个角色（带超时）"""
        try:
            start_time = asyncio.get_event_loop().time()
            
            # 调用AgentExecutor（需要异步化）
            response = await asyncio.wait_for(
                self._async_agent_execute(role_id, message),
                timeout=self.timeout
            )
            
            latency = (asyncio.get_event_loop().time() - start_time) * 1000
            return ScatterResult(
                role_id=role_id,
                success=True,
                response=response,
                latency_ms=int(latency),
                error=None
            )
        
        except asyncio.TimeoutError:
            return ScatterResult(
                role_id=role_id,
                success=False,
                response=None,
                latency_ms=self.timeout * 1000,
                error=f"Timeout after {self.timeout}s"
            )
        
        except Exception as e:
            return ScatterResult(
                role_id=role_id,
                success=False,
                response=None,
                latency_ms=0,
                error=str(e)
            )
```

#### 5.2.2 部分失败处理

```python
def aggregate_responses(self, results: List[ScatterResult]) -> str:
    """整合响应，处理部分失败"""
    
    successful = [r for r in results if r.success]
    failed = [r for r in results if not r.success]
    
    if not successful:
        # 全部失败：返回错误信息
        return self._format_all_failed(failed)
    
    if failed:
        # 部分失败：整合成功的，附加失败通知
        aggregated = self._merge_responses(successful)
        failure_note = self._format_partial_failure(failed)
        return f"{aggregated}\n\n⚠️ {failure_note}"
    
    # 全部成功：直接整合
    return self._merge_responses(successful)
```

```python
        self.max_rounds = max_rounds
        self.budget_tokens = budget_tokens
        self.enable_cycle_detection = enable_cycle_detection
    
    def allow_auto_routing(
        self, 
        context: RoutingContext
    ) -> tuple[bool, Optional[str]]:
        """
        判断是否允许自主路由
        
        Returns:
            (是否允许, 拒绝原因)
        """
        
        # 检查1：轮次限制
        if context.round_number >= self.max_rounds:
            return False, f"超过最大轮次限制 ({self.max_rounds})"
        
        # 检查2：预算限制
        if context.token_consumed >= self.budget_tokens:
            return False, f"超过token预算 ({self.budget_tokens})"
        
        # 检查3：循环检测
        if self.enable_cycle_detection:
            if self._detect_cycle(context.call_stack, context.target_role):
                return False, f"检测到调用循环: {' → '.join(context.call_stack)} → {context.target_role}"
        
        # 检查4：权限验证
        if not self._check_permission(context.source_role, context.target_role):
            return False, f"{context.source_role} 无权调用 {context.target_role}"
        
        return True, None
    
    def _detect_cycle(self, call_stack: List[str], target: str) -> bool:
        """检测调用栈中是否存在循环"""
        return target in call_stack
    
    def _check_permission(self, source: str, target: str) -> bool:
        """检查权限（从RoleConfig读取白名单/黑名单）"""
        role_config = RoleManager.get_role(source)
        
        # 黑名单优先
        if target in role_config.forbidden_targets:
            return False
        
        # 如果有白名单，必须在白名单中
        if role_config.allowed_targets:
            return target in role_config.allowed_targets
        
        # 无白名单：默认允许
        return True
```

```python
class ConversationContext:
    """对话上下文管理器"""
```

### 5.4 路由优先级冲突解决算法

#### 5.4.1 去重算法

```python
class MessageRouter:
    def deduplicate_targets(
        self, 
        explicit: List[str],
        auto: List[str],
        internal: List[str]
    ) -> List[tuple[str, RoutingMode]]:
        """
        去重并标记路由模式
        
        优先级: EXPLICIT > AUTO > INTERNAL
        """
        
        seen = set()
        result = []
        
        # Priority 1: 显式@
        for role_id in explicit:
            if role_id not in seen:
                result.append((role_id, RoutingMode.EXPLICIT))
                seen.add(role_id)
        
        # Priority 2: Coordinator分派
        for role_id in auto:
            if role_id not in seen:
                result.append((role_id, RoutingMode.AUTO))
                seen.add(role_id)
        
        # Priority 3: 自主路由
        for role_id in internal:
            if role_id not in seen:
                result.append((role_id, RoutingMode.INTERNAL))
                seen.add(role_id)
        
        return result
```
