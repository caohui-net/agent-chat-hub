## P1-001: Token预算控制契约

### 问题描述

**文件**: `src/agents/session.py:171`

**现状**:
```python
estimated_tokens = len(user_input) + len(response)  # 错误：使用字符串长度
```

**问题**:
- 使用字符串长度估算token，不准确
- 未使用供应商返回的真实usage
- 缺少token预算控制机制

### 最终契约（v2.0）

#### 1. Usage归一化规则

**统一接口定义**:
```python
@dataclass
class TokenUsage:
    """统一的Token使用量接口"""
    input_tokens: int      # 输入token数（包含缓存部分）
    output_tokens: int     # 输出token数
    cache_read_tokens: int # 缓存读取token数（不重复计入input）
    cache_write_tokens: int = 0  # 缓存写入token数（如果支持）
    
    @property
    def total_tokens(self) -> int:
        """总token数：input + output（cache_read已包含在input中）"""
        return self.input_tokens + self.output_tokens
```

**供应商映射规则**:

**Anthropic**:
```python
def _extract_anthropic_usage(api_response) -> TokenUsage:
    usage = api_response.usage
    return TokenUsage(
        input_tokens=usage.input_tokens,
        output_tokens=usage.output_tokens,
        cache_read_tokens=getattr(usage, 'cache_read_input_tokens', 0),
        cache_write_tokens=getattr(usage, 'cache_creation_input_tokens', 0)
    )
```

**OpenAI**:
```python
def _extract_openai_usage(api_response) -> TokenUsage:
    usage = api_response.usage
    # 注意：cached_tokens是prompt_tokens的子集，不重复计数
    return TokenUsage(
        input_tokens=usage.prompt_tokens,
        output_tokens=usage.completion_tokens,
        cache_read_tokens=getattr(usage, 'cached_tokens', 0),
        cache_write_tokens=0  # OpenAI暂不支持
    )
```

**关键约束**:
- ✅ `cache_read_tokens`已包含在`input_tokens`中，预算计算时**不重复累加**
- ✅ 预算计算公式：`total = input_tokens + output_tokens`
- ✅ 缺失usage时，标记为`unknown`或`estimated`，不使用字符串长度

#### 2. 并发预算控制策略

**策略选择**: **软限制**（Soft Limit）

**理由**:
- MVP阶段，单批次超额可接受
- 硬限制需要预留机制，实施复杂度高
- 会话级别累计控制足够

**实施方案**:
```python
class SessionManager:
    def __init__(self, token_budget: int):
        self.token_budget = token_budget
        self.tokens_used = 0
        self.max_overage_ratio = 0.2  # 最大超额20%
    
    async def run_round(self, user_input: str):
        # 软限制：并发前检查预算
        if self.tokens_used >= self.token_budget:
            raise BudgetExceededError(
                f"Session budget {self.token_budget} exceeded. "
                f"Used: {self.tokens_used}"
            )
        
        # 允许并发调用（可能超额）
        results = await asyncio.gather(
            *[agent.execute(user_input) for agent in qualified_agents]
        )
        
        # 事后统计并检查超额
        round_tokens = sum(agent.last_usage.total_tokens for agent in qualified_agents)
        self.tokens_used += round_tokens
        
        overage = self.tokens_used - self.token_budget
        if overage > 0:
            overage_ratio = overage / self.token_budget
            if overage_ratio > self.max_overage_ratio:
                logger.error(
                    f"Budget overage {overage_ratio:.1%} exceeds max {self.max_overage_ratio:.1%}"
                )
            else:
                logger.warning(f"Budget overage: {overage} tokens ({overage_ratio:.1%})")
```

**关键约束**:
- ✅ 单批次允许超额（所有并发调用都执行）
- ✅ 最大超额边界：预算的20%
- ✅ 超额后记录warning/error日志
- ✅ 会话级别累计，超额后拒绝新一轮调用

#### 3. Usage传递机制

**策略选择**: **MessageBus事件机制**

**理由**:
- 不破坏`AgentExecutor.execute`的字符串返回契约
- MessageBus已存在于会话中
- 事件机制天然支持异步
- 易于扩展（如添加成本计算、监控）

**实施方案**:

**Step 1: 定义Usage事件**:
```python
@dataclass
class AgentUsageEvent:
    """Agent Token使用事件"""
    event_type: str = "agent.usage"
    agent_id: str
    model_id: str
    usage: TokenUsage
    timestamp: float = field(default_factory=time.time)
```

**Step 2: AgentExecutor发布事件**:
```python
class AgentExecutor:
    async def _call_anthropic(self, messages, system_msg):
        response = await self.client.messages.create(...)
        
        # 提取usage
        usage = self._extract_anthropic_usage(response)
        
        # 发布usage事件（不改变返回值）
        await self.message_bus.publish(
            AgentUsageEvent(
                agent_id=self.agent_id,
                model_id=self.model_id,
                usage=usage
            )
        )
        
        # 保持字符串返回
        return response.content[0].text
```

**Step 3: SessionManager订阅事件**:
```python
class SessionManager:
    async def setup(self):
        # 订阅usage事件
        await self.message_bus.subscribe(
            'agent.usage',
            self._handle_usage_event
        )
    
    async def _handle_usage_event(self, event: AgentUsageEvent):
        # 累计token使用
        self.tokens_used += event.usage.total_tokens
        # 记录详细信息
        self.usage_history.append(event)
        # 可选：发布预算警告事件
        if self.tokens_used > self.token_budget * 0.8:
            await self.message_bus.publish(
                BudgetWarningEvent(used=self.tokens_used, budget=self.token_budget)
            )
```

**关键约束**:
- ✅ `AgentExecutor.execute`保持返回`str`
- ✅ Usage通过MessageBus异步传递
- ✅ 调用方无需修改（向后兼容）
- ✅ 失败调用返回`TokenUsage(0, 0, 0)`

### 验收标准

**测试覆盖**:
1. ✅ Anthropic usage提取（普通、缓存命中、缓存写入）
2. ✅ OpenAI usage提取（普通、缓存命中）
3. ✅ 缺失usage处理（标记unknown）
4. ✅ 并发调用超额检测（<20%, >20%）
5. ✅ MessageBus事件发布与订阅
6. ✅ 字符串返回值兼容性

**不变量**:
- `AgentExecutor.execute`签名不变：`async def execute(self, user_input: str) -> str`
- 预算计算公式一致：`total = input_tokens + output_tokens`
- 缓存token不重复计数

## P1-003: 模型引用验证契约

### 问题描述

**文件**: `src/core/config.py:145`

**现状**:
```python
def add_agent(self, agent: AgentConfig) -> None:
    self.agents[agent.agent_id] = agent  # 缺少：model_id验证
```

**问题**:
- add_agent时未验证model_id是否存在
- 模型删除/重命名/替换时未检查agent引用
- 配置加载时未验证引用完整性

### 最终契约（v2.0）

#### 1. 原子引用验证范围

**扩展到所有模型/Agent生命周期入口**:

**add_agent验证**:
```python
class SessionConfig:
    def add_agent(self, agent: AgentConfig) -> None:
        """添加Agent时验证model_id存在"""
        if agent.model_id not in self.models:
            raise ValueError(
                f"Cannot add agent '{agent.agent_id}': "
                f"model '{agent.model_id}' does not exist"
            )
        self.agents[agent.agent_id] = agent
```

**remove_model验证**:
```python
    def remove_model(self, model_id: str) -> None:
        """删除模型前检查是否有agent引用"""
        referencing_agents = [
            agent.agent_id
            for agent in self.agents.values()
            if agent.model_id == model_id
        ]
        
        if referencing_agents:
            raise ValueError(
                f"Cannot remove model '{model_id}': "
                f"referenced by agents: {', '.join(referencing_agents)}"
            )
        
        del self.models[model_id]
```

**update_model验证**:
```python
    def update_model(self, model_id: str, new_config: ModelConfig) -> None:
        """更新模型配置（model_id不可变）"""
        if model_id not in self.models:
            raise ValueError(f"Model '{model_id}' does not exist")
        
        # 禁止修改model_id（避免悬空引用）
        if new_config.model_id != model_id:
            raise ValueError(
                f"Cannot change model_id from '{model_id}' to '{new_config.model_id}'. "
                f"Use remove + add instead."
            )
        
        self.models[model_id] = new_config
```

**配置加载验证**:
```python
    @classmethod
    def load_from_file(cls, path: str) -> "SessionConfig":
        """加载配置时原子验证所有引用"""
        config = cls.parse_file(path)
        
        # 验证所有agent的model_id存在
        invalid_refs = []
        for agent in config.agents.values():
            if agent.model_id not in config.models:
                invalid_refs.append(
                    f"agent '{agent.agent_id}' -> missing model '{agent.model_id}'"
                )
        
        if invalid_refs:
            raise ValueError(
                f"Invalid configuration file '{path}':\n" +
                "\n".join(f"  - {ref}" for ref in invalid_refs)
            )
        
        return config
```

#### 2. 失败原子性

**约束**:
- ✅ 任何验证失败时，配置状态保持不变（原子性）
- ✅ 不允许部分成功的操作
- ✅ 错误信息明确指出违反的引用关系

**示例**:
```python
# 正确：删除模型前先删除引用的agent
config.remove_agent("agent1")
config.remove_model("model1")

# 错误：先删除模型会失败
config.remove_model("model1")  # ValueError: referenced by agents: ['agent1']
```

### 验收标准

**测试覆盖**:
1. ✅ add_agent验证：存在的model_id（成功），不存在的model_id（失败）
2. ✅ remove_model验证：无引用（成功），有引用（失败）
3. ✅ update_model验证：修改配置（成功），修改model_id（失败）
4. ✅ load_from_file验证：有效配置（成功），悬空引用（失败）
5. ✅ 失败后状态不变（原子性）

**不变量**:
- 任何时刻，所有agent.model_id都存在于config.models中
- 模型删除时，没有agent引用它
- 配置加载失败时，不创建SessionConfig实例

## P1-004: Provider支持契约

### 问题描述

**文件**: 
- `src/core/models.py` - ModelConfig定义
- `src/agents/executor.py:225` - AgentExecutor实现

**现状**:
```python
# ModelConfig接受4个provider
class ModelConfig(BaseModel):
    provider: Literal["anthropic", "openai", "google", "custom"]

# 但AgentExecutor只实现2个
class AgentExecutor:
    async def execute(self, user_input: str) -> str:
        if self.model_config.provider == "anthropic":
            return await self._call_anthropic(...)
        elif self.model_config.provider == "openai":
            return await self._call_openai(...)
        # google和custom未实现
```

**问题**:
- 配置允许的provider集合 > 执行支持的provider集合
- 用户配置google/custom后，运行时才发现不支持
- 历史配置可能包含google/custom

### 最终契约（v2.0）

#### 1. 分层策略：解析兼容 + 执行拦截

**理由**:
- 旧配置可能包含google/custom
- 直接收窄Pydantic枚举会导致配置加载失败（ValidationError）
- 需要优雅的迁移路径

**解析层（ModelConfig）**:
```python
class ModelConfig(BaseModel):
    """模型配置（解析层：兼容历史值）"""
    model_id: str
    provider: Literal["anthropic", "openai", "google", "custom"]
    # 保持接受所有历史值，不破坏配置加载
    
    model_config = ConfigDict(extra="forbid")
```

**执行层（AgentExecutor）**:
```python
class AgentExecutor:
    """Agent执行器（执行层：拦截不支持的provider）"""
    
    # MVP支持的provider集合
    SUPPORTED_PROVIDERS = {"anthropic", "openai"}
    
    async def execute(self, user_input: str) -> str:
        """执行Agent（拦截不支持的provider）"""
        model = self.session_config.models[self.model_id]
        
        # 执行前检查provider支持
        if model.provider not in self.SUPPORTED_PROVIDERS:
            raise UnsupportedProviderError(
                f"Provider '{model.provider}' is not supported in current version.\n"
                f"Supported providers: {', '.join(sorted(self.SUPPORTED_PROVIDERS))}\n"
                f"Please update your configuration to use a supported provider.\n"
                f"Migration guide: docs/provider-migration.md"
            )
        
        # 执行支持的provider
        if model.provider == "anthropic":
            return await self._call_anthropic(...)
        elif model.provider == "openai":
            return await self._call_openai(...)
        else:
            # 不应到达这里（已被上面的检查拦截）
            raise RuntimeError(f"Unreachable: provider={model.provider}")
```

**自定义异常**:
```python
class UnsupportedProviderError(Exception):
    """不支持的provider错误（可迁移）"""
    def __init__(self, message: str, provider: str = None):
        super().__init__(message)
        self.provider = provider
```

#### 2. 迁移策略

**配置验证工具**:
```python
def validate_config(config: SessionConfig) -> List[str]:
    """验证配置并返回警告列表"""
    warnings = []
    
    for model_id, model in config.models.items():
        if model.provider not in AgentExecutor.SUPPORTED_PROVIDERS:
            warnings.append(
                f"Model '{model_id}' uses unsupported provider '{model.provider}'. "
                f"This model will fail at runtime."
            )
    
    return warnings
```

**CLI提示**:
```bash
$ agent-chat-hub validate config.yaml

    This model will fail at runtime.
    Supported providers: anthropic, openai
    Migration guide: docs/provider-migration.md

✓ Configuration is valid (1 warning)
```

### 验收标准

**测试覆盖**:
1. ✅ 配置加载：google/custom配置可以被解析（不抛ValidationError）
2. ✅ 执行拦截：anthropic/openai执行成功，google/custom抛UnsupportedProviderError
3. ✅ 错误信息：包含明确的迁移指引
4. ✅ 验证工具：检测不支持的provider并给出警告

**不变量**:
- ModelConfig可以解析所有历史provider值（anthropic, openai, google, custom）
- AgentExecutor只执行SUPPORTED_PROVIDERS中的provider
- 不支持的provider在执行时抛出确定的、可迁移的错误（非运行时崩溃）
