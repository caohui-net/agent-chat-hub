## 问题1: Token预算控制完全失效

**编号**: P0-001  
**文件路径**: `src/agents/session.py`  
**行号**: 171  
**分类**: P0 - 功能阻塞

**问题代码**:
```python
# 执行成功
# 记录调用（估算token数）
estimated_tokens = len(user_input) + len(response)
self.coordinator.record_call(
    agent_config.agent_id,
    tokens_used=estimated_tokens
)
```

**预期行为**: 
- Token数应该使用实际的tokenizer计算（如tiktoken）
- 预算限制（12k tokens）应该准确反映实际API使用

**实际行为**:
- 使用字符串长度作为token估算
- 中文字符通常是2-4个tokens
- 英文单词平均1.3个tokens
- 误差可达300-500%

**影响范围**:
- 预算监控（BudgetLimits.max_tokens）完全失效
- 可能导致API成本失控
- coordinator.py的预算检查逻辑失效

**复现步骤**:
1. 创建session并发送中文用户输入："你好，请帮我分析这个问题"（13个字符）
2. Agent响应约100字符的中文文本
3. 代码估算：13 + 100 = 113 tokens
4. 实际使用：约300-500 tokens（使用tiktoken验证）
5. 预算监控误以为只用了113 tokens

**建议优先级**: P0
**原因**: 成本控制完全失效，生产环境风险高

**验证方法**:
```python
import tiktoken
encoding = tiktoken.get_encoding("cl100k_base")
actual_tokens = len(encoding.encode(user_input + response))
assert estimated_tokens != actual_tokens  # 将会失败
```

**建议修复**:
```python
# Option 1: 使用tiktoken
import tiktoken
encoding = tiktoken.get_encoding("cl100k_base")
estimated_tokens = len(encoding.encode(user_input)) + len(encoding.encode(response))

# Option 2: 从API响应中获取实际usage
# Anthropic/OpenAI API都会返回实际token使用量
```

## 问题2: Agent排序逻辑错误

**编号**: P0-002  
**文件路径**: `src/agents/coordinator.py`  
**行号**: 212  
**分类**: P0 - 功能阻塞

**问题代码**:
```python
def qualify_agents(
    self,
    available_agents: List[AgentConfig],
    max_agents: Optional[int] = None
) -> List[AgentConfig]:
    max_count = max_agents if max_agents is not None else self.budget_limits.max_agents
    
    # 过滤active agents
    qualified = [a for a in available_agents if a.active]
    
    # 限制数量（应用在排序后）
    return qualified[:max_count] if len(qualified) > max_count else qualified
```

**预期行为**:
根据ADR-0001规范，Ordering Rule要求：
1. 优先级升序（priority字段，数值越小越优先）
2. 相同优先级按agent_id字典序

**实际行为**:
1. 过滤active agents
2. **直接截断**前max_count个，未排序
3. 可能选择低优先级agents而忽略高优先级agents

**影响范围**:
- 违反ADR-0001的Ordering Rule
- Agent选择的确定性被破坏
- select_agents方法依赖此方法（line 272）

**复现步骤**:
1. 配置3个agents: 
   - agent_a (priority=100)
   - agent_b (priority=10)  
   - agent_c (priority=50)
2. 设置max_agents=2
3. 调用qualify_agents
4. 实际返回：[agent_a, agent_b]（列表顺序）
5. 预期返回：[agent_b, agent_c]（按priority排序后取前2个）

**建议优先级**: P0  
**原因**: 违反核心设计规范ADR-0001，影响agent选择的正确性

**验证方法**:
```python
def test_qualify_agents_ordering():
    coordinator = ResponseCoordinator()
    agents = [
        AgentConfig(agent_id="a", name="A", priority=100, model_id="m1"),
        AgentConfig(agent_id="b", name="B", priority=10, model_id="m1"),
        AgentConfig(agent_id="c", name="C", priority=50, model_id="m1"),
    ]
    qualified = coordinator.qualify_agents(agents, max_agents=2)
    assert qualified[0].agent_id == "b"  # 最低priority
    assert qualified[1].agent_id == "c"  # 第二低priority
```

**建议修复**:
```python
def qualify_agents(
    self,
    available_agents: List[AgentConfig],
    max_agents: Optional[int] = None
) -> List[AgentConfig]:
    max_count = max_agents if max_agents is not None else self.budget_limits.max_agents
    
    # 过滤active agents
    qualified = [a for a in available_agents if a.active]
    
    # 排序（调用sort_agents）
    sorted_agents = self.sort_agents(qualified)
    
    # 限制数量
    return sorted_agents[:max_count] if len(sorted_agents) > max_count else sorted_agents
```

## 问题4: 消息过滤逻辑不一致

**编号**: P1-002  
**文件路径**: `src/agents/executor.py`  
**行号**: 56, 149

**问题代码**:

    return [
        {"role": msg.role, "content": msg.content}
        for msg in messages
        if msg.role in ("user", "assistant")  # 过滤掉system
    ]

# Line 149: OpenAI实现又插入system消息

    api_messages = messages.copy()
    if system_prompt:
        api_messages.insert(0, {"role": "system", "content": system_prompt})

**预期行为**:

- 要么统一在_build_messages中处理system_prompt
- 要么在各provider方法中单独处理

**实际行为**:
- _build_messages过滤掉所有system消息
- _call_openai又插入system消息

- 逻辑不一致，容易产生bug

**影响范围**:

2. 调用executor.execute

## 问题5: Round计数不准确

**编号**: P1-003  
**文件路径**: `src/agents/session.py`  
**行号**: 269  
**分类**: P1 - 高优先级

**问题代码**:
```python
def load_session(self, session_id: str) -> SessionConfig:
    # ... 加载逻辑 ...
    
    self.current_session = session
    self.current_round = len([m for m in messages if m.role == "user"])
    # ❌ 问题：用user消息数量作为round数
```

**预期行为**:
- Round是对话的轮次，每次process_user_input是一轮
- 应该单独存储和恢复round_num

**实际行为**:
- 用user消息数量推测round数
- 但一轮可能有多个user消息（例如追问）
- 导致round计数不准确

**影响范围**:
- coordinator.start_round接收不准确的round_num
- 影响去重逻辑（CallRecord的三元组: session_id, round_num, agent_id）
- 可能导致同一agent在同一轮被重复调用

**复现步骤**:
1. 创建session，发送2个user消息在同一轮
2. 保存session
3. 加载session
4. current_round被计算为2（实际应该是1）
5. 新一轮开始时round_num=3（应该是2）

**建议优先级**: P1  
**原因**: 影响coordinator的去重逻辑，可能导致重复调用

**验证方法**:
```python
def test_round_counting():
    session.process_user_input("Hello")
    assert session.current_round == 1
    session.save_session()
    
    # 重新加载
    loaded = SessionManager(...)
    loaded.load_session(session.session_id)
    assert loaded.current_round == 1  # 当前会失败
```

**建议修复**:
```python
# 在SessionConfig中添加current_round字段
class SessionConfig(BaseModel):
    # ... 现有字段 ...
    current_round: int = Field(default=0, description="当前轮次")

# 保存时存储
def save_session(self) -> None:
    session_dict = {
        # ... 现有字段 ...
        "current_round": self.current_round,
    }

# 加载时恢复
def load_session(self, session_id: str) -> SessionConfig:
    session = SessionConfig(**session_dict)
    self.current_round = session.current_round
```

## 问题6: 缺少模型引用验证

**编号**: P1-004  
**文件路径**: `src/core/config.py`  
**行号**: 145 (add_agent方法)  
**分类**: P1 - 高优先级

**问题代码**:
```python
def add_agent(self, agent: AgentConfig) -> None:
    """添加或更新Agent配置
    
    Args:
        agent: Agent配置对象
    """
    self.agents[agent.agent_id] = agent
    # ❌ 没有验证agent.model_id是否存在于self.models
```

**预期行为**:
- add_agent时验证model_id是否存在
- 如果model_id不存在，抛出ValueError

**实际行为**:
- 不进行验证
- Agent可能引用不存在的模型
- 运行时executor.execute才会失败（line 196-198）

**影响范围**:
- 配置错误延迟到运行时才发现
- 用户体验差

**复现步骤**:
1. 添加Agent配置，model_id="non-existent"
2. add_agent成功（不应该）
3. 尝试调用该agent
4. executor.execute失败: "模型配置不存在: non-existent"

**建议优先级**: P1  
**原因**: 配置验证应该在添加时进行，不应该延迟到运行时

**验证方法**:
```python
def test_add_agent_validates_model():
    config_mgr = ConfigManager()
    agent = AgentConfig(
        agent_id="test",
        name="Test",
        model_id="non-existent",  # 不存在的模型
        role="test"
    )
    with pytest.raises(ValueError, match="模型配置不存在"):
        config_mgr.add_agent(agent)  # 当前不会抛出异常
```

**建议修复**:
```python
def add_agent(self, agent: AgentConfig) -> None:
    """添加或更新Agent配置
    
    Args:
        agent: Agent配置对象
        
    Raises:
        ValueError: 如果model_id不存在
    """
    # 验证model_id存在
    if agent.model_id not in self.models:
        raise ValueError(f"模型配置不存在: {agent.model_id}")
    
    self.agents[agent.agent_id] = agent
```
